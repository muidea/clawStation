# ClawStation 安装与使用说明

## 1. 获取安装包

请从 GitHub Release 下载：

* `ClawStation-windows-x86_64-gnu.zip`

当前正式支持的产品交付物只有这一份预编译 zip 包。

## 2. 解压后应看到的文件

标准目录下至少包含：

* `clawstation.exe`
* `clawops.exe`
* `WebView2Loader.dll`

要求：

* `clawstation.exe` 与 `clawops.exe` 必须保留在同一目录
* 不要单独移动其中某一个文件

## 3. 运行前准备

启动前请确认：

* Windows 已安装 Microsoft Edge WebView2 Runtime
* 当前机器可以访问目标 OpenClaw 主机
* 你已经准备好 SSH 账号以及对应密码或私钥

## 4. 启动应用

1. 解压 zip 包
2. 双击运行 `clawstation.exe`
3. 首次启动后，按界面提示添加认证配置和实例

说明：

* `clawops.exe` 会在需要时由 `clawstation.exe` 自动拉起
* 一般不需要手工单独启动 `clawops.exe`

## 5. 基本使用流程

### 5.1 添加认证配置

首次使用时，先添加一条 SSH 认证配置：

1. 填写 SSH 用户
2. 选择密码或私钥方式
3. 保存认证信息

### 5.2 添加实例

你可以添加两类实例：

* 已有实例接入
* 托管实例

### 5.3 连接与访问

实例保存后，可以继续：

* 建立连接
* 打开原界面
* 使用聊天能力
* 查看状态和错误信息

## 6. 升级方式

升级时建议：

1. 关闭当前运行中的 `ClawStation`
2. 下载新的发布 zip
3. 解压到新的目录，或覆盖旧目录中的同名文件
4. 再次启动 `clawstation.exe`

## 7. 认证信息安全存储说明

`ClawStation` 本地保存的 SSH 密码、私钥和相关安全凭据，默认会使用“本地主密钥 + 当前机器平台 UUID”派生出的密钥进行加密保存。

这意味着：

* 仅复制配置目录中的密文文件到另一台机器，默认无法解密
* 在同一台机器上重装操作系统后，只要平台 UUID 保持不变，原有密文通常仍可解密
* 更换主板、重建虚拟机、修改虚拟机硬件 UUID 后，原有密文可能失效

当前支持的平台 UUID 来源如下：

* Windows：`Win32_ComputerSystemProduct.UUID`
* Linux：`/sys/class/dmi/id/product_uuid`
* macOS：`IOPlatformUUID`

注意：

* 如果当前环境无法提供有效的平台 UUID，敏感认证信息将无法继续加密保存
* 容器、部分云主机、部分虚拟机和个别老旧设备，可能拿不到稳定的平台 UUID
* 该机制用于“限制密文跨机器复制后被直接解密”，不等价于硬件安全模块

## 8. 常见问题

### 8.1 双击后无法启动

优先检查：

* 是否缺少 WebView2 Runtime
* zip 是否完整解压
* `clawstation.exe` 和 `clawops.exe` 是否仍在同一目录

### 8.2 无法连接目标实例

优先检查：

* SSH 地址和端口是否正确
* 认证信息是否正确
* 目标主机密钥是否已信任
* 目标机器上的 OpenClaw 是否正常运行

### 8.3 为什么 Release 页面还能看到 Source code

这是 GitHub 平台基于 tag 自动生成的源码快照，不是产品发布资产。

正式支持的交付物仍然只有：

* `ClawStation-windows-x86_64-gnu.zip`
