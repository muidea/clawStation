# ClawStation

`ClawStation` 是一个面向 OpenClaw 实例的桌面端管理工具，用来把原本分散、重复、依赖命令行的连接与运维动作，收敛成一套可视化、可追踪、可复用的日常工作流。

它适合需要频繁访问、接管、部署和维护 OpenClaw 环境的使用者。无论你管理的是测试环境、预发环境还是生产环境，都可以通过统一入口完成实例接入、连接访问、诊断排查和基础运维。

## 它能做什么

`ClawStation` 当前聚焦在实例维度的日常管理，主要能力包括：

* 统一管理多个 OpenClaw 实例
* 管理 SSH 认证信息，支持密码和私钥方式
* 接管已有实例，减少手工连接和校验步骤
* 创建并托管新实例，降低部署门槛
* 自动建立访问链路并打开原始界面
* 提供实例级状态反馈、诊断信息和日志查看
* 支持常见运维动作，如部署、探测、重启、回滚
* 支持配置导入、导出与恢复，便于迁移和备份

## 为什么使用 ClawStation

直接管理 OpenClaw 时，常见问题通常不是“不会做”，而是流程过于分散：

* 访问依赖手工 SSH 隧道，步骤多且容易出错
* 多实例并行使用时，端口、入口和登录态容易混淆
* 新部署和已有实例常常分属不同处理路径，缺少统一视图
* 故障排查时，连接、诊断、日志和操作记录不在同一个入口

`ClawStation` 的价值就在于把这些高频但重复的动作集中起来，让连接、访问、接管、运维和排查形成一个连续闭环，而不是散落在多个脚本、命令和临时记录里。

## 适用场景

你可以在这些场景中使用 `ClawStation`：

* 需要长期维护多个 OpenClaw 环境
* 希望把已有实例纳入统一管理
* 希望用桌面端完成常见部署与访问操作
* 希望减少 SSH、探测、日志查看等重复劳动
* 希望在出现问题时快速定位实例状态和失败原因

## 如何开始使用

当前正式提供的交付物为 Windows 预编译安装包：

* `ClawStation-windows-x86_64-gnu.zip`

开始使用时只需要：

1. 从 GitHub Release 下载发布包
2. 解压后运行 `clawstation.exe`
3. 添加认证配置
4. 添加或接管实例
5. 直接在界面中完成连接、访问和基础运维

更详细的安装与操作说明见：

* [安装与使用说明](./docs/使用说明.md)
* [Windows 发布与安装说明](./docs/Windows发布说明.md)

## 使用收益

如果你希望把 OpenClaw 的日常使用从“依赖命令行的人肉流程”升级成“可重复、可管理、可诊断的桌面化体验”，`ClawStation` 就是为这个目标准备的。

它不是单纯的启动器，也不是零散脚本集合，而是一套围绕实例管理、访问入口和基础运维组织起来的产品化工具。
